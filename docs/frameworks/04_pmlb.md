# Framework card, PMLB

**Status in this repo: SURVEYED, NOT USED.** This lab builds its own case registry from primary
sources rather than pulling a benchmark collection.

## What it is, and the one thing it does better

PMLB, the Penn Machine Learning Benchmarks, is a curated collection of datasets with a uniform format
and a Python and R accessor. For symbolic regression it matters because it is where the Feynman and
ODE-Strogatz ground-truth problems actually live: SRBench's ground-truth track is PMLB's `feynman_*`
and `strogatz_*` datasets.

The one thing it does better than the alternatives is availability. The canonical Feynman Symbolic
Regression Database page at MIT is dead (`space.mit.edu/home/tegmark/aifeynman.html` returns HTTP 404
as of 2026-07-22, and its redirect target also returns 404), while every PMLB Feynman dataset still
cites that dead URL in its `source:` field. Anyone wanting the Feynman equations today takes them
from PMLB or from SRSD.

## Licence, and whether this MIT repo may use it

MIT (`github.com/EpistasisLab/pmlb`, verified through the GitHub licence API). Compatible, usable,

## Install reality

`pip install pmlb`, or the CRAN package `pmlbr`. Pure Python accessor over data files fetched from
GitHub. No compiler, no GPU. The practical constraint is bandwidth and disk, not build tooling.

## Usage

The counts, verified directly from `pmlb/all_summary_stats.tsv` on `master` and re-counted from that
file on 2026-07-22, are what a user needs before planning a run:

| Slice | Count |
|---|---|
| All datasets | 450 |
| Regression | 271 |
| Classification | 179 |
| `feynman_*` regression | 119 (99 core plus 20 `feynman_test_*` bonus equations) |
| `strogatz_*` regression | 14 (7 two-state systems, 2 states each) |
| Regression without ground truth | 138 |

The published PMLB v1.0 paper reported 417 datasets (255 regression, 162 classification), so the
collection has grown since publication. A result quoted against "PMLB" without a commit or release
pin is therefore not reproducible, which is why the research recommends shipping a per-file SHA-256
manifest for any dataset a lab depends on.

## Applying it here

Not used. This repo's cases are declared in
[`data-pipeline/symlab/cases/registry.py`](../../data-pipeline/symlab/cases/registry.py) and fetched
by [`symlab/io/loaders.py`](../../data-pipeline/symlab/io/loaders.py) from their primary sources,
with the defects each source actually has recorded in the case pages rather than tidied away (see
[docs/cases.md](../cases.md)). The categories include physics ground truth, industrial process data,
mining and metallurgy, biology, environment and synthetic generators.

Two reasons the collection was not adopted:

1. The lab's stated gap is mining and metallurgy data, which PMLB does not carry.
2. The recovery claim this lab makes depends on knowing the provenance and the defects of every
   source. A curated redistribution adds a layer between the loader and the instrument, and the case
   pages are written to remove layers, not add them.

If an SRBench-comparable number is ever wanted here, PMLB is the correct source for the Feynman and
Strogatz problems and this card is the entry point.

## Caveats

- The Feynman datasets in PMLB were generated by uniform sampling of standardised values, and
  Matsubara et al. showed that this "did not adequately depict the original physical phenomena". SRSD
  exists specifically to fix the sampling ranges; a Feynman recovery rate measured on PMLB sampling is
  not the same measurement as one on SRSD sampling.
- The collection grows between releases, so "PMLB" is a moving target unless pinned.
- `UNVERIFIED`: which three of the 133 PMLB ground-truth datasets SRBench v1 excludes to reach its
  stated 130.
